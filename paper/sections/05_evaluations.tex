\subsection{RQ1) To what extent can similarity-weighted voting on the nearest neighbors (\votag) classify issue reports, and at what point does adding neighbors stop helping?}
\label{sec:rq1}

To study \votag's performance as a function of $K$, We evaluate it at K values 1 to 30 across both PS and PA settings. \Cref{fig:vtag-kcurve} plots macro F1 as a function of $k$ in both settings, while \cref{tab:eval-ps,tab:eval-pa} report \votag's peak per-class and macro F1 at the chosen plateau $k$ ($k{=}7$ for PS, $k{=}14$ for PA, leftmost column of each table).


% Points to drive home in this subsection:
%
% 1. VOTAG defines a non-LLM retrieval floor.
%    Pure similarity-weighted k-NN over the FAISS index reaches macro F1
%    ~0.60 (agnostic) and ~0.58 (project-specific mean) at the plateau, in
%    negligible compute (CPU, no LLM call, no training). Any LLM-based
%    approach must clear this floor to justify the cost it adds.
%
% 2. VOTAG's k-curve plateau anchors the RAGTAG k-grid.
%    The k=1..30 sweep climbs to ~k=6, then plateaus (agnostic ~k=13,
%    project-specific ~k=7). The chosen RAGTAG grid {0, 1, 3, 6, 9}
%    brackets the climb (k=1, 3, 6) plus the entry to the plateau (k=9).
%    The k-values are not arbitrary -- they are empirically informed by
%    where retrieval evidence stops adding value. Pre-empts the
%    "did you try k=15? k=30?" reviewer question.
%
% 3. The retrieval signal is real but bounded.
%    At its plateau, VOTAG closes a meaningful chunk of the gap to
%    fine-tuning (FT-PA ~0.71 vs VOTAG ~0.60 -- about 85% of the way) but
%    still trails RAGTAG by ~0.10 macro F1 on every model size. This sets
%    up the natural question that motivates RAGTAG: what does the LLM add
%    beyond retrieval?
%

% - Table or figure: VOTAG macro F1 by k, both settings (agnostic, PS).
%   Mark the plateau. k = 1..15 or 1..30.

%   0.005 of the maximum).
% - Per-project breakdown: std across the 11 projects at the plateau k.
%   The point is consistency, not a single-project artifact.
% - PA vs PS comparison for VOTAG: agnostic outperforms PS by ~0.02 at
%   plateau. Mild evidence that pooling helps even pure retrieval.

\label{sec:bug-bias}

% Points to drive home in this subsection (chart deferred):
% 1. Per-class recall asymmetry across all LLM-based methods: bug recall is
%    high, feature recall medium, question recall the floor.
% 2. The asymmetry is invariant to model scale (3B-32B all show it).
% 3. Retrieval (RAGTAG) partially rebalances by exposing the LLM to
%    explicitly-labeled examples; FT also rebalances via gradient updates.
% 4. VOTAG (no LLM, model-independent) shows a milder version of the
%    asymmetry that comes purely from retrieval geometry, not the LLM prior.
% 5. Connection to Aracena et al. 2025: they identified question-class
%    difficulty in fine-tuned models; we extend the diagnosis to retrieval
%    and zero-shot, and show it is a structural LLM property.
